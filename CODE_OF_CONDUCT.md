# Code of Conduct

Across Orchestrator is built for thoughtful, practical collaboration. We expect
maintainers, contributors, issue reporters, and community participants to help
keep the project respectful, focused, and safe for technical work.

## Expected Behavior

- Be respectful when discussing design choices, bugs, tradeoffs, and priorities.
- Keep feedback specific to the work, the evidence, and the user impact.
- Respect privacy. Do not post private data, credentials, logs with secrets,
  personal paths, or screenshots that reveal sensitive information.
- Follow the project's license, contribution, security, and dependency policies.

## Unacceptable Behavior

- Harassment, threats, hate speech, personal attacks, or repeated disruptive
  behavior.
- Publishing another person's private information without explicit permission.
- Sharing secrets, exploit details, or sensitive operational information in
  public channels.
- Intentionally submitting code, assets, data, or dependencies that violate
  third-party rights or license terms.

## Scope

This code of conduct applies to project repositories, issue trackers,
discussions, pull requests, release channels, and project-related community
spaces.

## Reporting

For security issues, follow [SECURITY.md](SECURITY.md). For conduct concerns,
contact the maintainers through the repository's private maintainer channel when
available. Reports should include the relevant links, dates, and context.
